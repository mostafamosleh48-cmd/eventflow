# AGENTS.md — Eventflow

Webhook-driven task processing pipeline (Zapier-like). TypeScript, Express, PostgreSQL, Docker.

## Project Structure

```
src/
  server.ts          # Express app entry point (API server)
  worker.ts          # Background job worker entry point
  routes/            # Express route handlers (pipelines, webhooks, jobs)
  actions/           # Processing action implementations
  services/          # Business logic (queue, delivery + retry)
  db/
    pool.ts          # PostgreSQL connection pool
    migrations/      # SQL migration files
  types/             # Shared TypeScript type definitions
  utils/             # Helpers and shared utilities
tests/               # Jest test files (mirrors src/ structure)
docs/PRD.md          # Product requirements document
docker-compose.yml
Dockerfile
```

## Build & Run Commands

```bash
npm install              # Install dependencies
npm run dev              # Development (with hot reload)
npm run build            # Build TypeScript
npm start                # Start production server
npm run worker           # Start worker process
npm run migrate          # Run database migrations
```

## Code Style Guidelines

### TypeScript Strictness
- Enable `strict: true` in tsconfig.json
- No `any` types — use `unknown` and narrow with type guards
- Always define explicit return types on exported functions
- Use `interface` for object shapes, `type` for unions/intersections

### Naming Conventions

| Element           | Convention          | Example                    |
|-------------------|---------------------|----------------------------|
| Files             | camelCase           | `transformJson.ts`         |
| Directories       | camelCase           | `deliveryAttempts/`        |
| Classes           | PascalCase          | `PipelineService`          |
| Interfaces/Types  | PascalCase          | `PipelineConfig`           |
| Functions/methods | camelCase           | `processWebhook()`         |
| Constants         | UPPER_SNAKE_CASE    | `MAX_RETRY_ATTEMPTS`       |
| Env variables     | UPPER_SNAKE_CASE    | `DATABASE_URL`             |
| DB columns        | snake_case          | `action_config`            |
| API fields (JSON) | snake_case          | `pipeline_id`              |

### Imports
- Use ES module syntax (`import`/`export`) — no `require()`
- Order: (1) node built-ins, (2) external packages, (3) internal modules
- Separate each group with a blank line
- Prefer named exports over default exports

```typescript
import { randomUUID } from 'node:crypto';

import express from 'express';
import PgBoss from 'pg-boss';

import { PipelineService } from '../services/pipeline';
import type { Pipeline } from '../types/pipeline';
```

### Error Handling
- Wrap async route handlers with a centralized error-catching middleware
- Throw typed errors (extend a base `AppError` class with `statusCode`)
- Never swallow errors silently — always log with context
- Use `try/catch` in worker jobs; store `error_message` on failure
- HTTP status codes: 400 (validation), 404 (not found), 500 (server)

```typescript
class AppError extends Error {
  constructor(
    message: string,
    public statusCode: number = 500,
    public isOperational: boolean = true
  ) {
    super(message);
  }
}
```

### Formatting (Prettier)
- 2-space indentation, single quotes, trailing commas (ES5)
- Semicolons required, print width 100, LF line endings

### Database
- Use parameterized queries — never interpolate values into SQL
- Migrations are incremental SQL files, never destructive in production
- UUIDs for all primary keys (generated by PostgreSQL `gen_random_uuid()`)
- Timestamps are `TIMESTAMPTZ` (always UTC)
- Use connection pooling via `pg.Pool`

### API Conventions
- Base path: `/api/v1`
- RESTful resource naming (plural nouns): `/pipelines`, `/jobs`
- Response bodies use `snake_case` field names (matching DB columns)
- Return `201` for creation, `202` for async acceptance, `204` for deletion
- Pagination via `limit` and `offset` query params

### Git Commits

Use Conventional Commits format:

```
feat: add transform_json processing action
fix: handle null payload in webhook ingestion
test: add integration tests for delivery retry
chore: update Docker base image to node 20
refactor: extract action runner to separate module
```

### Environment Variables
- All config via env vars (12-factor style)
- Never commit `.env` files — use `.env.example` as a template
- Required vars: `DATABASE_URL`, `PORT`, `NODE_ENV`
- Optional vars: `REDIS_URL`, `WORKER_CONCURRENCY`, `JOB_MAX_RETRIES`

### Testing Practices
- Co-locate unit tests in `tests/unit/`, integration tests in `tests/integration/`
- Name test files `<module>.test.ts`
- Integration tests use a dedicated test database (use `DATABASE_URL` override)
- Clean up test data between runs (transaction rollback or truncation)
- Mock external HTTP calls in unit tests; use real DB in integration tests
